  <files>ZenSocial/Views/FloatingPillButton.swift</files>
  <action>
    Remove the `expandsLeft` computed property entirely.

    Add a new computed property `expandedPillOffset: CGFloat`:

    ```swift
    private var expandedPillOffset: CGFloat {
        let screenWidth = UIScreen.main.bounds.width
        let halfExpanded = expandedPillWidth / 2  // 86
        let leftEdge = nav.pillPosition.x - halfExpanded
        let rightEdge = nav.pillPosition.x + halfExpanded

        if leftEdge < edgeMargin {
            return edgeMargin - leftEdge  // positive, shift right
        } else if rightEdge > screenWidth - edgeMargin {
            return (screenWidth - edgeMargin) - rightEdge  // negative, shift left
        } else {
            return 0  // centered, no offset needed
        }
    }
    ```

    In the body, replace the expanded pill offset (currently lines 32-34):
    ```swift
    // Old:
    .offset(x: expandsLeft
        ? -(expandedPillWidth / 2 - pillSize / 2)
        : (expandedPillWidth / 2 - pillSize / 2))

    // New:
    .offset(x: expandedPillOffset)
    ```

    The expanded pill Group is already centered on `nav.pillPosition` via `.position()`, so with offset 0 it naturally expands symmetrically. The offset only kicks in when the expanded pill would clip past `edgeMargin` on either side.
