    **navigate command:**
    ```
    navigate)
      PLATFORM="${2:?navigate requires platform (instagram|youtube|home)}"
      echo "==> Setting lastPlatform to '$PLATFORM'..."
      xcrun simctl spawn booted defaults write com.zensocial.app lastPlatform "$PLATFORM"
      echo "==> Terminating app..."
      xcrun simctl terminate booted "$BUNDLE_ID" 2>/dev/null || true
      sleep 0.5
      echo "==> Relaunching app..."
      xcrun simctl launch booted "$BUNDLE_ID"
      sleep 2
      TS=$(date +%H%M%S)
      OUT="$SCREENSHOT_DIR/screen-$TS.png"
      xcrun simctl io booted screenshot "$OUT"
      echo "$OUT"
      ;;
    ```

    Key: uses `xcrun simctl spawn booted defaults write` (not `simctl launch --console`) because @AppStorage("lastPlatform") reads from the app's UserDefaults domain. NavigationState.restoreLastPlatform() reads `lastPlatformRaw` on init — the relaunch triggers this automatically. Valid values: instagram, youtube, home (matches Screen.rawValue in NavigationState).

    **watch command:**
    ```
    watch)
      INTERVAL="${2:-3}"
      echo "==> Watching simulator (every ${INTERVAL}s). Ctrl+C to stop."
      while true; do
        TS=$(date +%H%M%S)
        OUT="$SCREENSHOT_DIR/screen-$TS.png"
        xcrun simctl io booted screenshot "$OUT" 2>/dev/null && echo "$OUT"
        sleep "$INTERVAL"
      done
      ;;
    ```

    # Skill: sim-inspect — iOS Simulator Workflow

    ## When to Use
    Use sim-inspect.sh whenever you need to build, screenshot, navigate, or monitor the ZenSocial iOS Simulator during testing or CSS/theme debugging.

    ## Commands

    ### build (default)
    Builds ZenSocial, installs, launches, and takes a screenshot.
    ```bash
    ./Scripts/sim-inspect.sh
    # or explicitly:
    ./Scripts/sim-inspect.sh build
    ```
    Use when: Starting a new test session or after Swift code changes.

    ### screenshot
    Takes a screenshot of the current simulator state.
    ```bash
    ./Scripts/sim-inspect.sh screenshot
    # Returns: /tmp/zensocial-sim/screen-HHMMSS.png
    ```
    Use when: App is already running and you want a snapshot.

    ### navigate
    Sets UserDefaults, terminates, and relaunches the app to a specific tab.
    ```bash
    ./Scripts/sim-inspect.sh navigate instagram
    ./Scripts/sim-inspect.sh navigate youtube
    ./Scripts/sim-inspect.sh navigate home
    # Returns: /tmp/zensocial-sim/screen-HHMMSS.png
    ```
    Use when: You need to switch tabs. Do NOT use tap/cliclick for navigation — Metal-rendered Simulator windows reject coordinate clicks reliably.

    **How it works:** Writes `lastPlatform` key to the app's UserDefaults via `xcrun simctl spawn booted defaults write com.zensocial.app lastPlatform <value>`. On relaunch, `NavigationState.restoreLastPlatform()` reads this key and navigates automatically. No code changes required.

    Valid platforms: `instagram`, `youtube`, `home`

    ### watch
    Loops screenshots at a fixed interval. Prints each file path. Ctrl+C to stop.
    ```bash
    ./Scripts/sim-inspect.sh watch        # every 3 seconds (default)
    ./Scripts/sim-inspect.sh watch 5      # every 5 seconds
    ```
    Use when: Iterating on CSS/JS injection — watch output while editing Scripts/ files, then rebuild to see changes.

    ### tap
    Taps at device logical coordinates (unreliable on Metal-rendered windows).
    ```bash
    ./Scripts/sim-inspect.sh tap <device_x> <device_y>
    ```
    Note: Use `navigate` instead for tab switching. Reserve tap for UI elements that cannot be reached via UserDefaults.

    ### swipe
    Swipes from one coordinate to another.
    ```bash
    ./Scripts/sim-inspect.sh swipe <x> <y> [dx] [dy]
    ```

    ### winpos
    Prints Simulator window position and pre-calculated screen coordinates for common tap targets.
    ```bash
    ./Scripts/sim-inspect.sh winpos
    ```

    - Device ID: `15796420-57AA-4732-BE47-5AB2F98B7626` (iPhone 17)
    - Bundle ID: `com.zensocial.app`
    - Screenshots: `/tmp/zensocial-sim/screen-HHMMSS.png`
    - Script location: `Scripts/sim-inspect.sh` (project root)
