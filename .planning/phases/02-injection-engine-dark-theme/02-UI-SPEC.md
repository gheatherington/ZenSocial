# Phase 2 — UI Design Contract

> Visual and interaction contract for the CSS injection dark theme applied to Instagram and YouTube web content inside WKWebView. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Note:** This phase produces no new native SwiftUI screens. The "UI" is injected CSS that re-themes third-party web content. The design contract defines the exact color values, typography overrides, and visual rules the injected stylesheets must follow. One native UI element is added: an alert dialog for script-loading failures in release builds (D-10).

## Typography

Phase 2 does NOT override fonts or font sizes on Instagram or YouTube. Platforms use their own type system and altering it risks breaking layouts and losing the native-platform feel.

The injected CSS overrides **text color only** (see Color section below). Font family, font size, font weight, and line height are left untouched.

For the native error alert (D-10), SwiftUI system typography applies:

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Alert title | System .headline (17pt) | semibold (600) | System default |
| Alert message | System .subheadline (15pt) | regular (400) | System default |
| Alert button | System .body (17pt) | semibold (600) | System default |

## Color

### Injected CSS Theme Palette

All values sourced from `Color+ZenSocial.swift` (canonical reference from 02-CONTEXT.md).

| Role | Hex | RGB | CSS Variable Name | Usage |
|------|-----|-----|-------------------|-------|
| Dominant surface (60%) | #000000 | 0, 0, 0 | `--zen-bg-primary` | Page background, main content area background |
| Secondary surface (30%) | #1C1C1E | 28, 28, 30 | `--zen-bg-secondary` | Cards, navigation bars, toolbars, sidebars, modal backgrounds, input fields |
| Accent (10%) | #4DA6FF | 77, 166, 255 | `--zen-accent` | Links, active tab indicators, focus rings, selected state highlights, toggle-on states |
| Primary text | #FFFFFF | 255, 255, 255 | `--zen-text-primary` | Headings, usernames, primary labels |
| Secondary text | #8E8E93 | 142, 142, 147 | `--zen-text-secondary` | Timestamps, metadata, placeholder text, captions |
| Divider / border | #2C2C2E | 44, 44, 46 | `--zen-border` | Separator lines, input borders, card borders |
| Destructive | #FF3B30 | 255, 59, 48 | `--zen-destructive` | Error states only (not used in theme CSS; reserved for native alert) |

### Accent Reserved For (explicit list)

- Hyperlinks and tappable text links
- Active/selected tab indicator in platform navigation
- Focus rings on form inputs
- Toggle/switch active state
- "Follow" / primary action button backgrounds on Instagram
- Selected filter/chip highlight

### Color Application Rules

1. **`#000000` pure black for page backgrounds** — not `#1C1C1E`. This creates depth contrast between the page surface and secondary elements.
2. **`#1C1C1E` for elevated surfaces** — nav bars, toolbars, story trays, cards, modals, dropdowns, search bars.
3. **`#4DA6FF` accent applied sparingly** — only to interactive elements listed above. Never to static text, backgrounds, or decorative elements.
4. **Photos, video thumbnails, avatars, and user-generated content are NEVER re-themed** (D-02). CSS selectors must exclude `<img>`, `<video>`, `<canvas>`, and known content container classes.
5. **Platform brand colors preserved where appropriate** — Instagram gradient on stories ring, YouTube red on subscribe buttons. Do not override these with zenAccent.
6. **`prefers-color-scheme` is ignored** (D-04). CSS uses direct property overrides, not media queries.

## CSS Architecture Contract

### File Structure

```
ZenSocial/
  Scripts/
    Instagram/
      theme.css          -- Instagram dark theme overrides
    YouTube/
      theme.css          -- YouTube dark theme overrides
```

### Injection Rules

| Rule | Value | Source |
|------|-------|--------|
| Injection timing | `atDocumentStart` | D-08 — prevents FOUC |
| Injection method | CSS wrapped in JS: `document.documentElement.style` or `<style>` element prepended to `<head>` | CLAUDE.md pattern |
| Scope | `forMainFrameOnly: false` | Theme must apply to iframes (Instagram uses iframes for some modals) |
| Persistence | Automatic — WKUserScript persists across SPA navigations | 02-CONTEXT.md code_context |

### CSS Specificity Strategy

1. **First pass: CSS variable overrides** in `:root {}` block — target Instagram `--ig-*` variables and YouTube `--yt-spec-*` variables
2. **Second pass: Direct property overrides** with `!important` on elements that do not respond to variable overrides
3. **Selectors target structural elements only** — never content containers, media, or user-generated markup
4. **All overrides use `!important`** on direct property rules to beat platform specificity

### Push Notification Safety Constraints (D-11)

The injected CSS/JS MUST NOT:
- Block or redirect requests to service worker paths (`/sw.js`, `serviceworker`)
- Strip or modify `<link rel="manifest">` elements
- Override or disable `navigator.serviceWorker`, `Notification`, or `PushManager`
- Use broad wildcard selectors that could accidentally match service worker registration elements

## Interaction Contract

### Theme Loading Success (happy path)

1. WKWebView begins loading platform URL
2. `atDocumentStart` fires before first paint
3. CSS theme is injected into document
4. User sees dark-themed content from first visible frame (no FOUC)
5. Theme persists through all SPA navigations without re-injection

### Theme Loading Failure (error path — D-09, D-10)

**Debug build:**
1. Script bundle file not found at expected path
2. `assertionFailure` fires immediately — crash with clear message
3. Developer fixes build configuration

**Release build:**
1. Script bundle file not found at expected path
2. Injection skipped for failed script
3. Failure logged via `os_log` with subsystem and category
4. Native SwiftUI alert presented: "Theme failed to load"
5. User taps "Dismiss" to close alert, or "Report Issue" to open share sheet
6. Share sheet pre-populated with diagnostic text
7. Platform loads with its default (light) theme — fully functional, just unthemed

### No User-Initiated Interactions

Phase 2 has zero user-initiated interactions related to theming. The theme is always on (D-04, D-05). There is no toggle, no settings, no user choice. The only interaction is the error alert dismiss/report flow.
