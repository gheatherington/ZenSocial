Create three files:

**1. `ZenSocial/Services/ScriptLoader.swift`** -- A `@MainActor enum ScriptLoader` (stateless, no instances) that:

- Has a private static `Logger` (subsystem: `"com.zensocial.app"`, category: `"ScriptLoader"`)
- Exposes `static func themeScript(for platform: Platform) -> WKUserScript?`
  - Loads `theme.css` from `Scripts/{platform.displayName}/` subdirectory via `Bundle.main.url(forResource: "theme", withExtension: "css", subdirectory: "Scripts/\(platform.displayName)")`
  - Reads file contents as UTF-8 string
  - Calls `wrapCSSInJS(_:)` to wrap CSS in a JS IIFE
  - Returns `WKUserScript(source: js, injectionTime: .atDocumentStart, forMainFrameOnly: false)` per D-08
  - Returns `nil` on failure after calling `handleLoadFailure`
- Has `private static func wrapCSSInJS(_ css: String) -> String`
  - Escapes backslashes, backticks, and `$` signs for JS template literal safety
  - Returns an IIFE that creates a `<style>` element and appends it to `document.documentElement` (NOT `document.head` -- it is null at atDocumentStart)
  - The IIFE pattern: `(function() { var s = document.createElement('style'); s.textContent = \`{escaped_css}\`; document.documentElement.appendChild(s); })();`
- Has `private static func handleLoadFailure(platform: Platform, filename: String)`
  - Per D-09: In DEBUG builds, calls `assertionFailure` with a descriptive message including the expected path
  - Per D-10: In release builds, logs via `logger.error` and posts `NotificationCenter.default.post(name: .zenScriptLoadFailure, ...)` with userInfo containing `"platform"` (String) and `"filename"` (String)
- Defines the notification name: `extension Notification.Name { static let zenScriptLoadFailure = Notification.Name("zenScriptLoadFailure") }`

**Import only:** `WebKit`, `os`, `Foundation`

**Per D-11 (push notification safety):** The injected JavaScript is purely CSS injection -- it creates a `<style>` element and nothing else. It does NOT touch `navigator.serviceWorker`, `Notification`, `PushManager`, `<link rel="manifest">`, or any service worker paths. This is inherently safe because CSS cannot interfere with service workers.
