  <action>
Add the D-10 error alert to PlatformTabView. This alert only fires in release builds when ScriptLoader posts a `.zenScriptLoadFailure` notification.

Add state properties to `PlatformTabView`:
```swift
@State private var showScriptError = false
@State private var failedPlatform = ""
@State private var failedFilename = ""
```

Add an `.onReceive` modifier listening for the `.zenScriptLoadFailure` notification. Filter to only show the alert if the notification's `"platform"` matches `self.platform.displayName` (so each tab only shows its own failure):
```swift
.onReceive(NotificationCenter.default.publisher(for: .zenScriptLoadFailure)) { notification in
    guard let platform = notification.userInfo?["platform"] as? String,
          platform == self.platform.displayName else { return }
    failedPlatform = platform
    failedFilename = notification.userInfo?["filename"] as? String ?? "unknown"
    showScriptError = true
}
```

Add an `.alert` modifier using the exact UI-SPEC copywriting contract:
- Title: `"Theme failed to load"`
- Message: `"\(failedPlatform) theme could not be applied. The app will work normally without it."`
- "Dismiss" button (role: `.cancel`) -- closes the alert
- "Report Issue" button -- constructs diagnostic text and presents a share sheet via `UIActivityViewController`

The diagnostic text format (per UI-SPEC):
```
ZenSocial theme loading failed
Platform: {failedPlatform}
iOS: {UIDevice.current.systemVersion}
App: {Bundle.main.infoDictionary?["CFBundleShortVersionString"] ?? "unknown"}
Script: {failedFilename}
```

For presenting the share sheet from SwiftUI, use the standard pattern:
```swift
if let windowScene = UIApplication.shared.connectedScenes.first as? UIWindowScene,
   let rootVC = windowScene.windows.first?.rootViewController {
    let activityVC = UIActivityViewController(
        activityItems: [diagnosticText],
        applicationActivities: nil
    )
    rootVC.present(activityVC, animated: true)
}
```

Do NOT modify the existing error handling (ErrorView, loading overlay, auth modal). Only ADD the script failure alert alongside existing modifiers.

Add `import UIKit` if not already imported (needed for UIActivityViewController and UIDevice).
