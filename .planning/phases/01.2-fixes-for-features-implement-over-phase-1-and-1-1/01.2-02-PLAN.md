    **Fix the drag release teleport (D-05, D-06):**

    The current `dragGesture.onEnded` does:
    1. Computes `newPosition` from `nav.pillPosition + translation`
    2. Sets `dragOffset = .zero` immediately (teleport -- visual jump)
    3. Sets `nav.pillPosition = clampedPosition(newPosition)` (no animation)

    Replace the `onEnded` closure with a spring-animated position update:

    ```swift
    .onEnded { value in
        let newPosition = CGPoint(
            x: nav.pillPosition.x + value.translation.width,
            y: nav.pillPosition.y + value.translation.height
        )
        let clamped = clampedPosition(newPosition)

        // Animate the pill from its current visual position (pillPosition + dragOffset)
        // to the clamped final position. First, move pillPosition to where the pill
        // visually is right now (unclamped), then animate to clamped.
        nav.pillPosition = CGPoint(
            x: nav.pillPosition.x + value.translation.width,
            y: nav.pillPosition.y + value.translation.height
        )
        dragOffset = .zero

        // Spring animate to clamped position (D-06)
        if reduceMotion {
            nav.pillPosition = clamped
        } else {
            withAnimation(.spring(response: 0.35, dampingFraction: 0.8)) {
                nav.pillPosition = clamped
            }
        }
        nav.savePillPosition()
    }
    ```

    This approach:
    - Sets `nav.pillPosition` to the raw (unclamped) drop point and clears `dragOffset` in the same frame -- no visual jump because `position + offset` equals the new `position + 0`.
    - Then animates from the unclamped position to the clamped position with a spring -- smooth settle, not a teleport.
    - The `savePillPosition()` call saves the clamped position (the animation target). Since `withAnimation` sets the final value immediately for state purposes, `savePillPosition()` will persist the clamped value.

    **D-05 compliance:** Movement during drag is already 1:1 (`dragOffset = value.translation`). Do NOT change `.onChanged`. No animation during drag.

    **Fix expanded pill overflow (D-07, D-08):**

    The expanded pill is an HStack with 3 items (~164pt wide based on `3 * 44 + 2 * 8 + 2 * 12 = 180`). Currently it is centered on `nav.pillPosition` via `.position()`. When the pill is near the right edge, the right half clips off-screen.

    **Step 1: Add a computed property for expansion direction.**

    Add a private computed property to `FloatingPillButton`:

    ```swift
    /// Whether the expanded pill should expand leftward (pill is on right side of screen)
    private var expandsLeft: Bool {
        nav.pillPosition.x > UIScreen.main.bounds.width / 2
    }
    ```

    **Step 2: Offset the expanded pill based on direction.**

    The expanded pill is approximately 180pt wide. When centered on `pillPosition`, it extends 90pt in each direction. To prevent overflow:
    - If `expandsLeft` (pill on right half): shift the expanded pill leftward so its right edge aligns near the pill center
    - If expanding right (pill on left half): shift rightward so its left edge aligns near the pill center

    Modify the `expandedPill` view to apply a horizontal offset. After the `.shadow(...)` modifier on `expandedPill`, or by wrapping it:

    In the `body`, replace the Group that contains the if/else for expanded/collapsed:

    ```swift
    Group {
        if nav.isPillExpanded {
            expandedPill
                .offset(x: expandsLeft ? -(expandedPillWidth / 2 - pillSize / 2) : (expandedPillWidth / 2 - pillSize / 2))
        } else {
            collapsedPill
        }
    }
    .position(nav.pillPosition)
    .offset(dragOffset)
    ```

    Add a constant for the expanded pill width at the top with the other constants:
    ```swift
    private let expandedPillWidth: CGFloat = 180  // 3 items * 44pt + 2 * 8pt spacing + 2 * 12pt padding
    ```

    The offset formula `expandedPillWidth / 2 - pillSize / 2` shifts the expanded pill so the collapsed pill's center stays anchored to `pillPosition`, and the expansion happens in the opposite direction:
    - `expandsLeft`: offset = `-(180/2 - 56/2)` = `-62` (shifts left)
    - `expandsRight`: offset = `+(180/2 - 56/2)` = `+62` (shifts right)

    This way the pill "grows" from the collapsed circle toward the center of the screen, never off the edge.

    The actual expanded pill width is: `HStack(spacing: 8)` with 3 buttons of `frame(width: 44)` = `3*44 + 2*8 = 148`, plus `.padding(.horizontal, 12)` = `148 + 24 = 172`. Round up to 180 for safety margin. If after testing it seems off, adjust the constant.
