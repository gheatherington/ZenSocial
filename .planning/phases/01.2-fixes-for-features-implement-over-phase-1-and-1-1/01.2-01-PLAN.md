  <action>
    **Step 1: Expose WKWebView reference on WebViewState** (per D-01)

    In `WebViewState.swift`, add an `@ObservationIgnored` property to hold the WKWebView reference. This is NOT an observable property (we don't want view updates when it changes):

    ```swift
    import WebKit  // add this import

    @ObservationIgnored
    weak var webView: WKWebView? = nil
    ```

    Also add a convenience method for video pausing:

    ```swift
    func pauseAllVideos() {
        webView?.evaluateJavaScript("document.querySelectorAll('video').forEach(v => v.pause())", completionHandler: nil)
    }
    ```

    **Step 2: Set the webView reference in WebViewCoordinator**

    In `WebViewCoordinator.swift`, at the end of the `init` method (or wherever `coordinator.webView = webView` is set), also set `state.webView = webView`. Actually, the coordinator's webView is set in `PlatformWebView.makeUIView` via `context.coordinator.webView = webView`. Add a line right after that in `PlatformWebView.swift` -- but that file is not in this plan's ownership. Instead, use a `didSet` on the coordinator's `webView` property.

    In `WebViewCoordinator.swift`, change `weak var webView: WKWebView?` to:
    ```swift
    weak var webView: WKWebView? {
        didSet {
            state.webView = webView
        }
    }
    ```

    **Step 3: Pause videos on platform switch in ContentView** (per D-01, D-02)

    In `ContentView.swift`, add an `.onChange(of: nav.activePlatform)` modifier to the outer ZStack (after `.preferredColorScheme(.dark)`):

    ```swift
    .onChange(of: nav.activePlatform) { oldValue, newValue in
        // Pause videos on the platform being left (D-01, D-02)
        if oldValue == .instagram {
            instagramState.pauseAllVideos()
        }
        if oldValue == .youtube {
            youtubeState.pauseAllVideos()
        }
    }
    ```

    Note: `activePlatform` is a computed property. For `.onChange` to work, it needs to be derived from `activeScreen` which IS observable. If `.onChange(of: nav.activePlatform)` does not trigger (because computed properties don't publish changes), use `.onChange(of: nav.activeScreen)` instead and map to platform manually:

    ```swift
    .onChange(of: nav.activeScreen) { oldValue, _ in
        // Pause videos on the platform being left (D-01, D-02)
        switch oldValue {
        case .instagram:
            instagramState.pauseAllVideos()
        case .youtube:
            youtubeState.pauseAllVideos()
        case .home:
            break
        }
    }
    ```

    Use the `.onChange(of: nav.activeScreen)` variant -- it is more reliable because `activeScreen` is a stored `@Observable` property.
