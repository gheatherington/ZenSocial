# Phase 1 — UI Design Contract

> Visual and interaction contract for the native iOS shell. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (native iOS) |
| Preset | not applicable |
| Component library | SwiftUI + UIKit (UIViewRepresentable for WKWebView) |
| Icon library | SF Symbols (Apple system icons) |
| Font | SF Pro (iOS system font via `.body`, `.headline`, etc.) |

**Note:** This is a native iOS app. No web design system applies. All native UI uses SwiftUI with iOS system defaults. The design contract specifies values in iOS-native terms (Dynamic Type styles, system colors, SF Symbols).

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (1pt @1x) | Icon-to-label gaps within tab bar items |
| sm | 8px | Compact padding within error screen elements |
| md | 16px | Default content padding, error screen insets |
| lg | 24px | Spacing between error icon and error text |
| xl | 32px | Vertical spacing between error text and retry button |
| 2xl | 48px | Top padding for error screen content from center |
| 3xl | 64px | Not used in Phase 1 |

Exceptions: Tab bar uses iOS system-default sizing (49pt standard height). Pull-to-refresh uses system UIRefreshControl spacing. These are not customizable.

---

## Typography

Phase 1 uses iOS Dynamic Type styles exclusively. No custom fonts.

| Role | iOS Style | Size (default) | Weight | Line Height |
|------|-----------|----------------|--------|-------------|
| Body | `.body` | 17px | Regular (400) | 1.29 (22px) |
| Label | `.footnote` | 13px | Regular (400) | 1.38 (18px) |
| Heading | `.title2` | 22px | Bold (700) | 1.27 (28px) |
| Subheading | `.subheadline` | 15px | Regular (400) | 1.33 (20px) |

**Rationale:** Dynamic Type ensures accessibility compliance and respects user font size preferences. All text in the native shell (tab labels, error messages, loading text) uses these system styles. No custom font sizes.

## Color

ZenSocial's brand is "dark, minimalist." The native shell establishes this from Phase 1.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | System Background Dark (`#000000` in dark mode) | App background behind WKWebView, error screen background |
| Secondary (30%) | Secondary System Background (`#1C1C1E` in dark mode) | Error screen card surface, tab bar background |
| Accent (10%) | `#4DA6FF` (calm blue) | Retry button on error screen, active tab icon tint |
| Destructive | System Red (`#FF453A` in dark mode) | Not used in Phase 1 |

Accent reserved for: active tab bar icon tint, retry button fill on error screen. Nothing else. Inactive tab icons use `#8E8E93` (system gray).

**Color mode:** Dark mode only for Phase 1. The app's `Info.plist` sets `UIUserInterfaceStyle` to `Dark`. This is a locked decision -- ZenSocial's entire brand is dark/minimal.

**System color mapping:**
- Tab bar background: `.ultraThinMaterial` over black (system default dark tab bar)
- Tab bar active icon: accent blue `#4DA6FF`
- Tab bar inactive icon: `#8E8E93` (systemGray)
- Loading indicator: system `ProgressView` uses accent blue
- Error screen background: primary black
- Error screen text: `#FFFFFF` (primary label)
- Error screen secondary text: `#EBEBF599` (secondary label, 60% white)

## Component Inventory

### 1. Tab Bar (SHELL-01)

**Component:** SwiftUI `TabView` with `.tabItem` modifiers

| Property | Value |
|----------|-------|
| Style | System default dark mode tab bar |
| Tab count | 2 (Instagram, YouTube) |
| Tab 1 icon | SF Symbol: `camera` (from PlatformConfig) |
| Tab 1 label | "Instagram" |
| Tab 2 icon | SF Symbol: `play.rectangle` (from PlatformConfig) |
| Tab 2 label | "YouTube" |
| Active tint | `#4DA6FF` (accent blue) |
| Inactive tint | System gray (`#8E8E93`) |
| Position | Bottom, system default |
| Background | `.ultraThinMaterial` (system blur) |

**Interaction:** Tapping a tab switches the visible WKWebView. Each tab retains its WKWebView state (scroll position, loaded page). Switching tabs does not reload the web view.

### 2. Loading Indicator (SHELL-02)

**Component:** SwiftUI `ProgressView` overlay

| Property | Value |
|----------|-------|
| Type | Indeterminate circular spinner (`ProgressView()`) |
| Position | Centered in web view area |
| Background | Semi-transparent black overlay (`Color.black.opacity(0.3)`) |
| Tint | Accent blue `#4DA6FF` |
| Visibility | Shown from `webView.isLoading == true` until first contentful paint |
| Animation | System default fade in/out (0.25s) |

**Interaction:** Loading indicator appears when a page starts loading. Disappears when `WKNavigationDelegate.didFinish` fires. No user interaction with the indicator. The web view content underneath is not interactive while loading overlay is visible.

### 3. Error / Offline Screen (SHELL-03)

**Component:** Custom SwiftUI view replacing web view content

| Property | Value |
|----------|-------|
| Layout | VStack, centered vertically and horizontally |
| Background | Primary black (`#000000`) |
| Icon | SF Symbol: `wifi.slash` (offline) or `exclamationmark.triangle` (generic error) |
| Icon size | 48pt |
| Icon color | System gray (`#8E8E93`) |
| Heading | `.title2` style, primary white |
| Body | `.subheadline` style, secondary label color |
| Retry button | Filled style, accent blue background, white label |
| Retry button padding | 16px horizontal, 12px vertical |
| Retry button corner radius | 10px (system default for medium buttons) |

**States:**

| State | Icon | Heading | Body |
|-------|------|---------|------|
| Offline | `wifi.slash` | "You're Offline" | "Check your internet connection and try again." |
| Load failed | `exclamationmark.triangle` | "Something Went Wrong" | "{Platform} couldn't load. Try again in a moment." |

**Interaction:** Tapping "Try Again" reloads the current platform URL via `webView.reload()`. Error screen replaces the web view area entirely (not an overlay). Tab bar remains visible and functional during error state.

### 4. Pull-to-Refresh (WEB-05)

**Component:** `UIRefreshControl` attached to WKWebView's scroll view

| Property | Value |
|----------|-------|
| Type | System `UIRefreshControl` |
| Tint | Accent blue `#4DA6FF` |
| Position | System default (above scroll content) |

**Interaction:** User pulls down on web content to trigger refresh. System spinner appears. Web view calls `webView.reload()`. Spinner dismisses when `didFinish` navigation delegate fires.

### 5. Web View Container (WEB-01, WEB-02)

**Component:** `PlatformWebView` (UIViewRepresentable wrapping WKWebView)

| Property | Value |
|----------|-------|
| Layout | Full screen minus tab bar (`.ignoresSafeArea(.all, edges: .top)`) |
| Background | Black (`#000000`) — prevents white flash during page load |
| Scroll | Native WKWebView scrolling |
| Back/forward | System swipe gestures (`webView.allowsBackForwardNavigationGestures = true`) (WEB-04) |

**Interaction:** Standard web browsing. Tap, scroll, swipe back/forward. All navigation within the platform domain is allowed. External links open in system browser.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Try Again" (error screen retry button) |
| Empty state heading | Not applicable — web view always loads a URL; no empty state in Phase 1 |
| Empty state body | Not applicable |
| Error state (offline) | "You're Offline" / "Check your internet connection and try again." |
| Error state (load failed) | "Something Went Wrong" / "{Platform} couldn't load. Try again in a moment." |
| Destructive confirmation | None in Phase 1 — no destructive actions |

**Copy guidelines:**
- Use sentence case for all UI text
- Platform names are proper nouns: "Instagram", "YouTube" (capitalized)
- Error copy is calm and non-technical — no error codes, no blame language
- Retry action uses "Try Again" (verb + adverb), not "Retry" or "Reload"

## State Machine

### Per-Tab Web View States

```
[idle] ---(tab selected)---> [loading]
[loading] ---(didFinish)---> [loaded]
[loading] ---(didFail + offline)---> [error:offline]
[loading] ---(didFail + other)---> [error:failed]
[loaded] ---(pull-to-refresh)---> [loading]
[loaded] ---(navigate)---> [loading]
[error:offline] ---(tap retry)---> [loading]
[error:failed] ---(tap retry)---> [loading]
[error:*] ---(tab switch + return)---> [error:*] (persists until retry)
```

Each tab maintains independent state. Switching tabs does not reset the other tab's state.

## Accessibility Contract

| Element | Requirement |
|---------|-------------|
| Tab bar | System-provided accessibility labels (automatic from `.tabItem`) |
| Loading indicator | `accessibilityLabel("Loading {Platform}")` |
| Error screen icon | Decorative — `accessibilityHidden(true)` |
| Error heading | Included in VoiceOver reading order |
| Error body | Included in VoiceOver reading order |
| Retry button | `accessibilityLabel("Try again")`, `accessibilityHint("Reloads {Platform}")` |
| Web view | System WKWebView accessibility (pass-through to web content) |
| Dynamic Type | All native text respects Dynamic Type scaling |
| Minimum touch target | 44x44pt for retry button (enforced via padding) |

## Animation Contract

| Element | Animation | Duration | Curve |
|---------|-----------|----------|-------|
| Loading overlay appear | Fade in | 0.25s | `.easeIn` |
| Loading overlay dismiss | Fade out | 0.25s | `.easeOut` |
| Error screen appear | No animation (immediate replacement) | 0s | n/a |
| Error screen dismiss | Fade out to web view | 0.3s | `.easeOut` |
| Tab switch | System default (immediate, no transition) | 0s | n/a |

No custom animations beyond these. The app should feel instant and calm — no bounces, no springs, no decorative motion.
