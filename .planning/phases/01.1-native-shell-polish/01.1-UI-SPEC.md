# Phase 01.1 — UI Design Contract

> Visual and interaction contract for the Native Shell Polish phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (native iOS) |
| Preset | not applicable |
| Component library | SwiftUI + UIKit (WKWebView wrapper) |
| Icon library | SF Symbols (Apple system) |
| Font | San Francisco (iOS system font) |

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4pt | Icon gaps, inline padding |
| sm | 8pt | Compact element spacing, text gaps |
| md | 16pt | Default element spacing, horizontal padding |
| lg | 24pt | Section padding, icon-to-text gaps |
| xl | 32pt | Layout gaps between content groups |
| 2xl | 48pt | Major section breaks on home screen |
| 3xl | 64pt | Page-level vertical margins |

Exceptions:
- Floating pill button collapsed size: 56pt diameter (touch target)
- Floating pill button expanded height: 56pt
- Pill item tap targets: 44pt minimum (iOS HIG minimum)
- Platform launcher cards: 120pt height
- Corner radius on pill: 28pt (half of 56pt height)
- Corner radius on cards: 16pt
- Corner radius on buttons: 8pt (existing ErrorView uses 10pt -- update to 8pt as part of this phase)

---

## Typography

All typography uses the iOS system font (San Francisco) via SwiftUI dynamic type styles. Point sizes below are base sizes before Dynamic Type scaling.

| Role | SwiftUI Style | Size | Weight | Line Height |
|------|---------------|------|--------|-------------|
| Body | `.body` | 17pt | Regular (400) | 1.5 (25pt) |
| Label | `.subheadline` | 15pt | Regular (400) | 1.4 (21pt) |
| Heading | `.title2` | 22pt | Bold (700) | 1.2 (26pt) |
| Display | `.largeTitle` | 34pt | Bold (700) | 1.2 (41pt) |

Weight usage:
- **Regular (400)**: Body text, labels, secondary copy
- **Bold (700)**: Headings, display text, button labels, pill item labels on long-press

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#000000` (pure black) | App background, WKWebView background, all full-screen surfaces |
| Secondary (30%) | `#1C1C1E` (`zenSecondaryBackground`) | Cards, floating pill background, loading screen surfaces, home screen cards |
| Accent (10%) | `#4DA6FF` (`zenAccent`) | See reserved list below |
| Destructive | `#FF453A` (iOS system red) | Destructive actions only (none in this phase) |
| Inactive | `#8E8E93` (`zenInactiveGray`) | Unselected pill items, secondary icons, disabled states |
| Text Primary | `#FFFFFF` | Headings, body text, active labels |
| Text Secondary | `#FFFFFF` at 60% opacity | Subheadlines, helper text, timestamps |

**Accent reserved for:**
- Active/selected pill item icon tint
- "Try Again" button background (error state)
- Pull-to-refresh spinner tint
- Platform launcher card accent border/highlight on tap
- Loading screen progress indicator
- Settings chevron/icon on home screen

**Platform identity colors** (used in loading screens and launcher cards as subtle tints, not as primary surfaces):

| Platform | Color | Hex | Usage |
|----------|-------|-----|-------|
| Instagram | Gradient pink-orange | `#E1306C` | Loading screen icon tint, launcher card accent stripe |
| YouTube | Red | `#FF0000` | Loading screen icon tint, launcher card accent stripe |

Platform colors appear only as icon tints and thin accent elements (4pt stripe on card edge). They never replace `zenAccent` for interactive elements.

## Component Inventory

### 1. Floating Pill Navigation Button

**Collapsed state:**
- 56pt diameter circle
- Background: `#1C1C1E` at 90% opacity with `.ultraThinMaterial` blur
- Icon: `house.fill` SF Symbol, 22pt, tinted `zenAccent`
- Shadow: 0pt x 4pt blur 12pt, black at 40% opacity
- Position: bottom-trailing corner, 16pt from safe area edges (default)
- Draggable: user can reposition; position persists in UserDefaults

**Expanded state:**
- Horizontal pill: height 56pt, width auto-sized to content
- Background: `#1C1C1E` at 95% opacity with `.ultraThinMaterial` blur
- Corner radius: 28pt (full pill shape)
- Contains 3 items in horizontal layout: Home, Instagram, YouTube
- Item spacing: 8pt between items, 12pt horizontal padding on edges
- Each item: 44pt tap target, SF Symbol icon at 20pt
- Active item: icon tinted `zenAccent`
- Inactive items: icon tinted `zenInactiveGray`
- Shadow: same as collapsed

**Expand/collapse animation:**
- Duration: 0.3 seconds
- Curve: `.spring(response: 0.35, dampingFraction: 0.8)`
- Collapse triggers: tap outside pill, or item selection

**Pill item icons:**
| Item | SF Symbol | Label (long-press only) |
|------|-----------|------------------------|
| Home | `house.fill` | "Home" |
| Instagram | `camera.fill` | "Instagram" |
| YouTube | `play.rectangle.fill` | "YouTube" |

### 2. Home Screen

**Layout:**
- Full-screen black background
- Content vertically centered with VStack, spacing 48pt
- ZenSocial branding at top: app name in `.largeTitle` bold, white
- Optional tagline: "Social media, without the noise." in `.subheadline`, text secondary color
- Platform launcher cards in VStack, spacing 16pt
- Settings entry at bottom of card group

**Platform Launcher Card:**
- Full width minus 32pt horizontal padding (16pt each side)
- Height: 120pt
- Background: `#1C1C1E`
- Corner radius: 16pt
- Left edge: 4pt wide vertical stripe in platform identity color
- Content: HStack with platform icon (SF Symbol, 28pt, platform color tint), platform name in `.title2` bold, chevron `chevron.right` in `zenInactiveGray` at trailing edge
- Padding: 24pt internal
- Tap: navigates to platform, pill collapses

**Settings Entry:**
- Same card style as platform launchers but no platform color stripe
- Icon: `gearshape.fill` SF Symbol, 28pt, `zenInactiveGray`
- Label: "Settings" in `.title2` bold
- Chevron at trailing edge
- Tap: navigates to placeholder settings screen

### 3. Context-Aware Loading Screens

**Layout:**
- Full-screen overlay on black background
- Content vertically centered
- VStack, spacing 24pt

**Variants:**

| Variant | Icon | Heading | Subheading |
|---------|------|---------|------------|
| Loading Instagram | `camera.fill` at 48pt, tinted `#E1306C` | "Loading Instagram" | "Just a moment..." |
| Loading YouTube | `play.rectangle.fill` at 48pt, tinted `#FF0000` | "Loading YouTube" | "Just a moment..." |
| Waiting for Sign In | `person.crop.circle` at 48pt, tinted `zenAccent` | "Waiting for Sign In" | "Complete login to continue." |
| General fallback | `globe` at 48pt, tinted `zenInactiveGray` | "Loading..." | "Just a moment..." |

- Heading: `.title2` bold, white
- Subheading: `.subheadline` regular, text secondary
- Progress indicator: indeterminate `ProgressView()` tinted `zenAccent`, placed below subheading with 16pt spacing

### 4. Auth Half-Sheet

**Presentation:**
- `.sheet(item: $state.pendingAuthURL)` binding preserved from Phase 1
- `.presentationDetents([.medium])` for half-sheet
- `.presentationDragIndicator(.visible)` — standard iOS grab handle
- Background dimming: default system behavior (web content visible and dimmed behind)

**Content:**
- WKWebView fills the sheet
- WKWebView background: black (already set in AuthModalView)
- No additional native chrome inside the sheet

### 5. Settings Placeholder Screen

**Layout:**
- NavigationStack with title "Settings"
- Full-screen black background
- Centered content: VStack, spacing 16pt
- Icon: `gearshape.fill` at 48pt, `zenInactiveGray`
- Text: "Settings coming soon" in `.body` regular, text secondary
- No interactive elements

### 6. Fade Transitions

**Between views (home to platform, platform to platform):**
- `.opacity` transition
- Duration: 0.25 seconds
- Curve: `.easeInOut`
- Web views remain in view hierarchy (ZStack-always-rendered pattern preserved) -- only opacity changes

## Interaction Contracts

### Drag-to-Reposition (Floating Pill)

- Standard iOS `DragGesture` on the collapsed pill button
- Bounds clamping: pill stays within safe area insets + 8pt margin
- Snap behavior: none (free positioning, stays where user drops it)
- Position saved to `UserDefaults` as CGPoint on drag end
- Restored on app launch; default position if no saved value: bottom-trailing, 16pt from edges

### Launch Behavior

- First launch (no `@AppStorage` value): show home screen
- Subsequent launches: restore last active platform from `@AppStorage("lastPlatform")`
- If restored platform's WebView is still loading: show loading screen for that platform

### Pill Expansion

- Tap collapsed pill: expand to horizontal dock with spring animation
- Tap pill item: navigate to target, collapse pill
- Tap outside pill: collapse pill
- Long-press pill item: show tooltip with item label (standard SwiftUI `.help()` modifier)

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| VoiceOver labels | All pill items, cards, and buttons have `.accessibilityLabel` |
| VoiceOver hints | Navigation items include `.accessibilityHint` describing destination |
| Dynamic Type | All text uses SwiftUI text styles (scales automatically) |
| Minimum tap targets | 44pt minimum on all interactive elements |
| Reduce Motion | Respect `UIAccessibility.isReduceMotionEnabled` — skip fade/spring animations |
| Color contrast | White text on black: 21:1 ratio (exceeds WCAG AAA). `zenAccent` on black: 5.2:1 (passes WCAG AA) |
